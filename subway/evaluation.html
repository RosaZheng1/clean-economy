Project evaluation process

There is a process that NRCan and CRA must follow when issuing a project evaluation.

Process

  1. Evaluation of project plan submission (NRCan)

    Evaluation process
    • Review of CCUS process, CCUS property, and whether the submitted project plan meets the definition of a CCUS project
    • Review of detailed technical documentation provided by the claimant to evaluate whether property is expected to be CCUS property, consistent with the technical guidance
    • Clarification or additional information and documentation may be requested if the submitted documentation is not sufficient to complete an evaluation

  2. Determination of project plan (CRA and NRCan)

    Determination process
    • Review and determination of whether a CCUS project is a single project or multiple projects
      • Refer to Factors considered in determining whether there is a single CCUS project or multiple CCUS projects
    • Project determination can occur at any time before the initial project evaluation has been issued
    • If a request that a revised project plan be submitted for the project, the project determination can occur any time after the revised project plan has been submitted and before the revised project evaluation is issued
    • For each project determined, a project plan must be submitted no later than 180 days after the determination is made

  3. Issuance of project evaluation (NRCan)

    Issuance process
    • Initial and revised project evaluations are uploaded in NRCan's secure portal
    • Claimants can review their project evaluation within the portal

NRCan and CRA responsibilities

NRCan responsibilities

NRCan is responsible for:

  • receiving project plans
  • issuing initial project evaluations
  • issuing revised project evaluations
  • verifying whether property acquired before the first day of commercial operations is CCUS property
  • collecting annual claim reports to verify whether property for which a CCUS Tax Credit is claimed is eligible CCUS property
  • collecting Knowledge Sharing reports
CRA responsibilities

CRA is responsible for determining whether a CCUS project constitutes as one project or multiple projects.

Initial project evaluation

An initial project evaluation must be completed prior to project construction to claim investment tax credits on a go-forward basis over the course of construction.

For projects that may already be under construction prior to claiming the investment tax credit, the issuance of an initial project evaluation would still be required before investment tax credit related to project construction could be claimed.

These could apply retroactively, back to eligible expenditures that were incurred starting on or after January 1, 2022.

Factors considered in determination of project plan

Factors considered in determining whether there is a single CCUS project or multiple CCUS projects.

When making its determination of what constitutes a single CCUS project or multiple CCUS projects, the CRA will, in consultation with NRCan, undertake an analysis of a number of relevant factors, including the ones listed below. This analysis and determination will be made on a case-by-case basis considering all the particular facts and circumstances.

In additional to the following factors, the CRA may also consider the presentation and description of the project(s) by a particular owner in their financial statements, and other public documents, as well as those presented to one or more regulatory and other government agencies (at all relevant levels of government), where applicable, to aid in the determination of whether there is a single CCUS project or multiple CCUS projects.

Ownership

One or more of the CCUS segments within a CCUS value chain may have different owners as opposed to a single owner for the entire CCUS value chain. Different owners of the CCUS segments comprising a CCUS value chain could indicate multiple CCUS projects.

Location

In some situations, the existence of multiple distinct project sites of the same nature, such as storage sites, capture sites, and CO2 emission sources sites, may suggest that there are multiple CCUS projects. However, where there is physical and/or functional integration of infrastructure and equipment among multiple project sites of the same nature, it may be considered to be a single CCUS project (for example, where multiple wells are drilled to faciliate the storage of CO2 from a common source).

Capacity

Differences in the capacity of different segments in a CCUS value chain could indicate multiple CCUS projects. For example, some CCUS value chains include transportation and storage or use segments that are built with the capacity to transport and store or use the CO2 from a single capture segment. This could indicate that all the CCUS segments in that CCUS value chain are part of a single CCUS project. Other CCUS value chains include CO2 transportation and/or storage or use segments that are specifically overbuilt to accommodate CO2 from multiple CO2 capture segments (i.e., they have larger CO2 capacity than the aggregate of the initially-connected CO2 capture segments), such as a trunk line or a storage hub. This fact could indicate that the CO2 transportation and/or storage or use segments are not part of the same CCUS project(s) as the individual CO2 capture segments. This may also be the case if, for example, a use segment has lower CO2 capacity than the trunk line that it is connected to. In that situation, this fact could indicate that the CO2 use segment is a separate CCUS project from the other CCUS segments in the CCUS value-chain.

Phased construction

If the construction of the facilities and the infrastructure of one or more CCUS segments in a CCUS value chain occurs in phases, the CRA may consider a phase to be a separate CCUS project, if for example, such phase was not included and specified in the initial project plan or such phase begins or is under construction after the project start-up date of a different phase of the CCUS segment(s).

Commencement of operations of CCUS segments in a CCUS value chain

Where the project start-up date of a particular CCUS capture segment, coincides with the project start-up date of another CCUS segment in the same CCUS value chain, such as a capture segment, this could suggest that the CCUS segments in that particular CCUS value chain are part of the same CCUS project.

Significant expansion of capacity

If an existing CCUS project that has commenced commercial operations undergoes a significant expansion of its capacity, for example the addition of a new CO2 trunk line or storage hub, it could be considered to be a new CCUS project.

Written agreements

To the extent that there are written agreements in place between the relevant participants in a particular CCUS value chain, such as partnership agreements, joint venture agreements, or a contract for services or letter of intent between the owners of the CCUS capture and the CCUS transportation and/or storage and use segments, these could be examined to ascertain the degree of interconnection and interdependence between the CCUS segments in order to determine whether there is a single CCUS project or multiple CCUS projects.
